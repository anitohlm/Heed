# Technical Doc Screenshots

The Word doc generated by `generate_azure_doc.py` references screenshots
from this folder. If a file is missing the doc still builds — a styled
placeholder appears in its place — but for the polished version drop the
PNGs in here using the exact filenames below, then re-run:

```bash
python generate_azure_doc.py
```

## Filenames expected (Section 10)

| Filename | What it should show |
|---|---|
| `10-old-app-y1-with-eol-banner.png` | `func-heed` Function App overview, plan SKU "Dynamic Y1", with the Microsoft "Migrate your app to Flex Consumption…" banner at the top. |
| `10-flex-app-essentials.png` | The new `func-heed-flex` Overview pane after migration: Plan type **Flex Consumption**, Operating System Linux, "Functions" list empty / "Create functions in your preferred environment". |
| `10-publish-output-with-invoke-urls.png` | Terminal output of `.\functions\deploy_functions.ps1` after `--build remote`. The "Functions in func-heed-flex:" block listing every function with its **Invoke url** (the URL field is the source of truth, not the function name). |
| `10-cors-configured.png` | Output of `az functionapp cors add …` showing `allowedOrigins` populated with the SWA origin, localhost, and `*`. |
| `10-flex-log-stream-success.png` | Function App → Monitoring → Log stream during a successful chat round-trip. Lines like `Executing 'Functions.advisor_stream'` and `Executed … (Succeeded, Duration=…ms)`, plus the chained Azure OpenAI `200 OK` requests. |

## Filename conventions

- Lowercase, hyphen-separated.
- Prefix with the section number (`10-…`) so future sections can be added without renames.
- PNG format, full screenshot or relevant region — the generator scales to 6 inches wide by default.
